About Portfolio Resume

During my time at RSI, I worked on two major projects. The first was the Internal Resumes project and the second was a web automating script.

PROJECT: INTERNAL RESUMES

Rural Sourcing, Inc (RSI) provides specialized IT services for companies and thus is continually developing new software applications. In order to ensure that employees are put on projects that match their current skill levels, they are encouraged to update their internal resumes frequently. The current process of updating their internal resumes and searching through internal resumes is a very manual and tedious one. The web app that I worked on aimed to reduce the amount of manual labor it takes to create, maintain, and search through internal resumes.

PROCESS

As projects develop, there are bound to be major changes. The internal resumes project started out as an individual project where I was the only person working on it, but changed midway into a more complicated application that required a team of people to work on it. I will explain my role both prior and post the project change.

Individual Project:
Originally, I created a web form that was hosted on the cloud through Amazon Web Services. The backend was developed in Python using Flask as its framework. The frontend was developed using the Bootstrap framework to maintain consistency with RSI’s other internal applications. The information entered into the web form was stored into a MySQL database. Team Project:
The project shifted from a basic web form into a more complicated user portal system with a focus on being able to manage and search for employees by “skill.” Basically, the scope of the web application expanded way beyond just a web form. As the new team was forming, I was tasked with the responsibility of creating UI wireframes for the portal. Working closely with the client, I used Balsamiq to iteratively design the UI features and functionality.

PROJECT: WEB AUTOMATION

RSI originally had a system that downloaded SQL files from OpenAir and then uploaded those files to their SQL database once a day at around 4 am. The problem with this was that there was no way to update the data on demand. If you needed the most up to date data for a client, say, in the afternoon, the current system would not provide the most updated data. The web automating script I wrote allowed RSI to run the script at any given time and thus can provide the most current information when needed.

RESULT

The script was my first introduction in web automation. I learned just how powerful it can be and how much time large-scale companies save by automating processes. While short, one script can have a huge impact. By creating our own in-house web automating process for this specific task, I was able to save the company around $5000 in licences. While it’s a small amount on its own, it’s the little items that add up and can create a huge difference.
